# 정해진 파일 외 건들일 땐 팀원과 소통 후 변경
# 파일을 추가할 때도 팀원과 소통

# 알아야 하는 것들 모아놓은 텍스트 파일


project-root/
│
├── app.py                 # Flask 실행 메인 파일
├── requirements.txt       # 설치할 패키지 목록(모듈, 패키지 추가할때마다 여기에 추가바람!!)
├── info.txt               # 우리가 알고있으면 좋은 정보 모음
├── .gitignore             # 깃 제외 항목
│
│
├── routes/                # 각 기능의 라우터
│   ├──__init__.py         # 블루프린트 모음
│   ├── auth_routes.py     # 로그인/회원가입 관련 라우터 처리
│   └──...# 등등
│
├── services/              # 핵심 로직 처리 (DB 저장, 필터링 등)
│   ├── config.py          # mongoDB uri 정보
│   ├── auth_service.py    # 로그인/회원가입 관련 서버 처리
│   ├── ...# 등등
│   └──...# 등등
│
├── static/                # CSS, JS 등 정적 파일
├── templates/             # HTML 파일
│
├── docker-compose.yml     # 서비스 및 컨테이너 오케스트레이션 설정
├── Dockerfile             # 컨테이너 빌드 설정
├── .github/                # GitHub Actions 워크플로우 설정


run.py	모든 Blueprint 등록 및 서버 실행
routes/	요청 URL과 기능 연결 (예: /login 처리)
services/ 실제 로직 처리 (DB 저장, 처리 알고리즘 등)
static/, templates/	프론트엔드 요소
